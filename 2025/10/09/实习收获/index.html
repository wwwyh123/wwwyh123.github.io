  • git commit –amend 覆盖最后一次commit

  • Husky 触发 pre-commit
  • 执行 npx lint-staged

数据的流式获取
HTTP 分块流式传输（Chunked Transfer Encoding）特点：基于 HTTP/1.1，服务器将响应内容分成多个“块”逐步发送给客户端。单向通信：服务器 → 客户端。数据格式：客户端需自行解析分块数据（如 JSON 流）。优点：无需额外协议支持，兼容性好。适合动态生成内容（如 AI 流式输出、日志实时传输）。缺点：需手动处理分块逻辑。无法实现双向通信。示例场景：AI 模型生成文本时的 token 级流式输出。大文件下载（如视频、日志文件）。 WebsocketWebSocket 协议主要位于 OSI 七层模型的应用层（第7层），就像HTTP一样。不过，WebSocket 的创建是通过一个HTTP请求来初始化的，在初始握手阶段之后，它会升级为WebSocket连接。因此，在连接建立初期，它涉及到了传输层（通常是TCP）的使用，但一旦连接建立，WebSocket 就直接运行在TCP之上，而不经过HTTP协议栈了。webSocket...
