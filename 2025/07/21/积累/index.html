setTimeout(1000) 一定是 1000ms 吗？（定时不准原因）

  1. 主线程阻塞（最常见原因）
    如果主线程正在执行耗时任务（如大量计算、复杂 DOM 操作），即使 setTimeout 的时间到了，回调也必须排队等待主线程空闲。
  2. 最小延迟限制（浏览器/系统限制）
    浏览器对 setTimeout 有最小延迟限制，通常为 4ms（即使你写 setTimeout(fn, 0)）。
    在嵌套调用或低优先级任务中，可能被延长到 10ms 或更多。
  3. 其他宏任务竞争
    JavaScript 的事件循环中，setTimeout 回调属于宏任务（macrotask），它需要和其他宏任务（如 setInterval、I/O、DOM事件）竞争执行时机。

事件循环：

浏览器中的 JavaScript 只有一个主线程，负责执行代码、处理用户交互和渲染页面。如果直接在主线程执行耗时操作（如定时器、网络请求），会导致页面“卡顿”或“阻塞”。
事件循环（Event Loop）的作用：通过 异步任务调度 和 任务队列管理，让 JavaScript 在单线程环境下高效处理并发任务（如 I/O 操作、定时器、用户事件等），避免阻塞主线程。

  1. 同步任务优先：所有同步代码会立即执行，阻塞后续代码直到调用栈清空。
  2. 微任务优先级更高：微任务队列会在每次宏任务结束后立即清空，优先于下一个宏任务。
  3. 异步任务分类：
  • 宏任务：setTimeout、setInterval、I/O、UI 渲染等。
  • 微任务：Promise.then、MutationObserver、queueMicrotask 等。

请你介绍一下vue

Vue 是一个用于构建用户界面的 渐进式 JavaScript 框架，由尤雨溪开发。

渐进式”意味着你可以根据项目需要，逐步采用 Vue 的功能，而不是“全有或全无”。意味着你可以从一个简单的 CDN 引入开始，只用它的模板语法做数据绑定；也可以逐步引入 Vue Router、Vuex/Pinia、Vite 等生态工具，构建大型单页应用（SPA）。这种灵活性让它非常适合从小项目到企业级应用的过渡。”

Vue 的核心特性包括：

  • 响应式系统：基于 Proxy（Vue3）实现数据劫持，数据变化自动更新视图，开发者无需手动操作 DOM。
  • 组件化开发：通过 .vue 单文件组件（SFC），把模板、逻辑、样式封装在一起，提升复用性和可维护性。
  • 虚拟 DOM：通过 render 函数生成 VNode，结合 diff 算法高效更新真实 DOM，提升性能。
  • 组合式 API（Composition API）：在 Vue3 中引入，相比选项式 API，它让逻辑复用更灵活（比如用 useMouse、useFetch 等自定义 Hook），特别适合复杂组件的逻辑拆分。
    丰富的生态系统：Vue Router 做路由，Pinia 做状态管理（推荐替代 Vuex），Vite 做极速构建，还有 Nuxt.js 做服务端渲染。”
    “我在项目中使用 Vue3 + Composition API + TypeScript + Vite 的技术栈，比如做过一个后台管理系统，通过 ref、reactive 管理状态，用 onMounted 发起请求，computed 处理表格数据，watch 监听搜索条件变化。还用 defineComponent 和 Prop 类型定义组件接口，提升了代码可读性和类型安全。”

Vue 不只是一个框架，更是一种渐进式构建用户界面的思维方式，它让开发者既能快速交付，又能优雅演进

Vue 的依赖收集和更新机制是其响应式系统的核心

它使得数据变化能够自动触发视图的更新。这个过程主要涉及以下几个关键概念：数据劫持、依赖收集、派发更新。

  1. 数据劫持（Data Interception）
    Vue 2 使用 Object.defineProperty 来劫持对象的 getter 和 setter，而 Vue 3 则使用了更强大的 Proxy。当一个 Vue 实例被创建时，它会遍历 data 对象的所有属性，并使用这些方法将它们转换为响应式的。

Vue 2: 通过 Object.defineProperty 为每个属性定义 getter 和 setter。
Vue 3: 使用 Proxy 包装整个对象，可以监听到对象属性的添加和删除等操作。
2. 依赖收集（Dependency Collection）
当组件渲染时，会读取 data 中的数据，这时就会触发对应属性的 getter。

在 getter 中，Vue 会进行依赖收集，即将当前正在渲染的组件（或 watcher）作为依赖，添加到该属性的依赖列表中。
每个响应式属性都有一个 Dep（Dependency）实例，用来存储所有依赖于它的 watchers。
这个过程发生在组件的渲染阶段，即首次执行 render 函数时。
3. 派发更新（Notify Updates）
当数据发生变化时，会触发 setter。

在 setter 中，Vue 会通知 Dep 实例，调用 dep.notify()。
Dep 会遍历其依赖列表（即所有 watcher），并执行它们的 update 方法。
每个 watcher 的 update 方法会重新执行组件的 render 函数，从而更新视图。

浏览器拿到html之后怎么解析、渲染的

  1. 解析 HTML，构建 DOM 树
    浏览器从服务器接收到 HTML 字节流后，首先进行字节解析（Bytes → Characters），根据响应头中的字符编码（如 UTF-8）将字节转换为文本。
    然后进行词法分析（Lexical Analysis） 和 语法分析（Parsing），将 HTML 文本解析成一个个标签（tokens），并构建出 DOM（Document Object Model）树。
    DOM 是一个 JavaScript 对象表示的树形结构，代表了页面的结构。
    📌 示例：<div><p>Hello</p></div> 会被解析为嵌套的 DOM 节点。

  2. 遇到 CSS，构建 CSSOM
    当解析 HTML 时，如果遇到 <link rel="stylesheet"> 或 <style>标签，浏览器会阻塞 DOM 构建（除非是 media=”print” 等非关键媒体类型），去加载并解析 CSS。
    CSS 被解析成 CSSOM（CSS Object Model），它是一个树状结构，包含了所有 CSS 选择器和规则。
    CSSOM 是阻塞渲染的：因为样式会影响布局，浏览器必须等 CSSOM 构建完成才能进行后续渲染。
    ⚠️ 所以：CSS 被称为“渲染阻塞资源”。

  3. 构建渲染树（Render Tree）
    浏览器将 DOM 和 CSSOM 合并成一个 渲染树（Render Tree）。
    渲染树只包含需要显示的节点（例如不包括 display: none 的元素，但包括 visibility: hidden 的元素）。
    每个渲染树节点（称为“渲染对象”或“box”）都包含了其对应的 DOM 节点和计算后的样式。

  4. 布局（Layout / Reflow）
    又称“重排”，浏览器计算每个渲染树节点在设备视口中的几何位置和大小。
    这个过程是递归的：从根元素（）开始，为每个可见元素确定位置和尺寸。
    布局的结果是一棵树，其中每个节点都有明确的坐标（如 x, y, width, height）。
    💡 布局是开销很大的操作，应尽量避免频繁触发。

  5. 绘制（Paint / Rasterization）
    又称“重绘”，浏览器将渲染树的每个节点转换为屏幕上的像素。
    这个过程包括：
    将节点拆分为多个图层（Layer）
    对每个图层进行光栅化（Rasterization）：将矢量图形转换为位图（像素）
    有些部分可能在 GPU 上进行合成（Compositing）
    🎨 绘制包括文本、颜色、边框、阴影等视觉属性。

  6. 合成与显示（Compositing & Display）
    现代浏览器使用分层机制：将页面分成多个图层（如固定背景、动画元素等），分别光栅化。
    合成器线程（Compositor Thread） 将这些图层合并成一个最终图像，发送给 GPU 渲染到屏幕上。
    这样可以高效处理动画和滚动：只需更新变化的图层，而不必重绘整个页面。

  1. 调用方式：构造函数：可以用 new 调用，也可以普通调用（虽然不推荐），Class：必须通过 new 调用，否则抛出错误。
  2. 方法的可枚举性：
  • 构造函数原型方法：默认可枚举。Object.keys(Person.prototype) // ['sayHello']（如果手动添加）
  • Class 方法：不可枚举。底层原因：class 中定义的方法通过 Object.defineProperty 添加，enumerable: false。
  1. prototype 属性的可写性
  • 构造函数：prototype 可以被重新赋值。Person.prototype = { ... };
  • Class：prototype 属性不可写（writable: false）。
  1. 类名提升（Hoisting）与暂时性死区（TDZ）
  • 构造函数：函数声明会提升，可以在声明前使用
  • Class：class 声明不会提升，存在暂时性死区
  1. 静态方法的实现:
  • 构造函数：手动添加到函数本身。Person.create = function() { return new Person('Anonymous'); };
  • Class：使用 static 关键字。

  • 构造函数继承：
  • class继承 实现原理：

HTTPS 加密的核心原理

HTTPS并不是一种全新的协议，而是HTTP + SSL/TLS。它的加密过程结合了对称加密和非对称加密的优点，通过一个称为 TLS 握手（TLS Handshake） 的过程来建立安全连接。

一、核心加密技术

  1. 非对称加密（Asymmetric Encryption）
    使用一对密钥：公钥（Public Key） 和 私钥（Private Key）
    公钥可以公开，私钥必须保密
    用公钥加密的数据，只能用私钥解密
    用私钥签名的数据，可以用公钥验证
    ✅ 用途：身份认证、密钥交换

❌ 缺点：计算复杂，速度慢，不适合加密大量数据

  1. 对称加密（Symmetric Encryption）
    加密和解密使用同一个密钥
    速度快，适合加密大量数据
    ✅ 用途：加密实际传输的数据（如网页内容）

二、HTTPS 是如何工作的？（TLS 握手过程）
我们以访问 https://www.example.com 为例：

  • 步骤 1：客户端发起连接（Client Hello）
    客户端（浏览器）向服务器发送：

    • 支持的 TLS 版本
    • 支持的加密套件（Cipher Suites）
    • 一个随机数（Client Random）

  • 步骤 2：服务器响应（Server Hello）
    服务器选择一个双方都支持的加密套件和 TLS 版本
    返回：

    • 服务器的数字证书（包含公钥、域名、颁发机构等）
    • 一个随机数（Server Random）
    • 会话 ID（可选）

  • 步骤 3：验证证书
    浏览器检查证书是否有效：

    • 是否由可信的 CA（证书颁发机构）签发
    • 域名是否匹配
    • 是否在有效期内
    • 是否被吊销
      如果证书无效，浏览器会警告用户

  • 步骤 4：生成“预主密钥”（Pre-Master Secret）
    客户端生成一个随机的预主密钥（Pre-Master Secret）
    用服务器的公钥加密这个预主密钥
    发送给服务器
    ✅ 只有服务器能用私钥解密，获取预主密钥

  • 步骤 5：生成“会话密钥”（Session Key）
    客户端和服务器使用：
    Client Random
    Server Random
    Pre-Master Secret
    通过相同的算法生成一个对称的会话密钥（Session Key）
    🔑 这个密钥用于后续通信的对称加密

  • 步骤 6：切换加密通信
    双方发送“加密切换”通知
    之后的所有通信都使用对称加密（如 AES）和这个会话密钥

重排,重绘

✅ 定义：
重排（也叫“回流”）是指浏览器重新计算页面中元素的几何信息（位置、大小、布局）的过程。

📌 触发条件（任何改变布局的操作）：

  • 添加、删除 DOM 元素
  • 修改几何属性：width, height, margin, padding, border
  • 改变字体：font-size, line-height
  • 调整窗口大小（resize）
  • 计算 offsetTop, offsetWidth 等布局信息（强制同步重排）
  • 激活 CSS 伪类（如 :hover）
  • 内容变化（如文本或图片加载）
    ⚠️ 特点：
  • 开销巨大：重排会触发整个渲染树或部分树的重新计算。
  • 连锁反应：一个元素重排，可能导致其子元素、兄弟元素甚至整个页面重排。
    什么是重绘（Repaint）？
    ✅ 定义：
    重绘 是指浏览器重新绘制元素的外观（颜色、背景、边框样式等），但不改变布局。

📌 触发条件（改变视觉样式但不影响布局）：

  • 修改颜色：color, background-color
  • 修改边框样式：border-style, visibility
  • 修改文本样式：text-decoration, font-family,outline、box-shadow 等
    ⚠️ 特点：
    比重排轻量，但仍有一定性能开销。
    不涉及位置和大小的重新计算。

如何减少重排和重绘？

  1. 批量修改 DOM
    避免频繁操作 DOM，先用 documentFragment 或 innerHTML 批量更新。
  2. 使用 transform 替代几何属性
    transform 属于合成阶段，不触发重排或重绘（在 GPU 上处理）。
  3. 使用 position: fixed 或 absolute
    将频繁动画的元素脱离文档流，减少对其他元素的影响。
  4. 避免使用表格布局
    表格的布局算法复杂，小改动可能导致整个表格重排。

js作用域

作用域就是变量和函数的“可见范围”。
JavaScript 采用的是 词法作用域，也叫静态作用域 —— 作用域在代码写好时就确定了，不是运行时决定的。

  1. 全局作用域（Global Scope）
  • 在任何函数或块外部定义的变量，属于全局作用域。
  • 全局变量可以在代码的任何地方被访问
  1. 函数作用域（Function Scope）
  • 在函数内部定义的变量，只能在该函数内部访问。
  • 每次调用函数，都会创建一个新的函数作用域。
  1. 块级作用域（Block Scope）（ES6 新增）
  • 使用 let 和 const 在 {} 内部定义的变量，只能在该块内访问。
  • 常见于 if、for、while 等语句块中。

作用域链（Scope Chain）

当 JS 查找一个变量时，会按照以下顺序查找：

  • 当前作用域
  • 外层函数作用域
  • 再外层函数作用域
    …
  • 全局作用域
  • 如果还没找到，报错：ReferenceError

WeakSet 和 WeakMap 是 JavaScript 中提供的两种特殊的集合类型，它们与 Set 和 Map 类似，但在内存管理和对象引用方面有着不同的行为。了解为什么需要 WeakSet 和 WeakMap 需要从它们的特点以及使用场景来探讨。

主要特点
弱引用：WeakSet 和 WeakMap 存储的键（对于 WeakMap 而言）或值（对于 WeakSet 而言）是对象的弱引用。这意味着，如果一个对象仅被这些集合引用，它仍然可以被垃圾回收器回收。
不可迭代：与 Set 和 Map 不同，WeakSet 和 WeakMap 不能被遍历（例如，没有 .entries()、.keys() 或 .values() 方法），也不提供获取所有键或值的方法。
键的特殊性：在 WeakMap 中，键必须是对象，而在 WeakSet 中，只能存储对象。
为什么需要 WeakSet 和 WeakMap
避免内存泄漏
当你使用 Set 或 Map 来保存对某些对象的引用时，只要这个 Set 或 Map 存在，即使这些对象在其他地方不再被使用，它们也不会被垃圾回收器回收。这可能导致内存泄漏。而使用 WeakSet 和 WeakMap，当对象不再被其他地方引用时，它可以被垃圾回收器自动回收。
临时数据关联
WeakMap 特别适用于当你想将一些额外的数据关联到某个对象上，但又不希望影响该对象的生命周期。例如，在实现缓存或者元数据管理时非常有用。
私有数据机制
WeakMap 可以用来创建某种形式的“私有”数据结构。由于其键是弱引用的对象，且无法枚举，因此可以安全地用于存储不应该暴露给外部代码的信息。
