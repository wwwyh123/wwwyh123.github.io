String.prototype.replace()

replace(pattern,replacement) 方法返回一个新字符串，其中一个、多个或所有匹配的 pattern 被替换为 replacement。

  • pattern 可以是字符串或 RegExp，replacement 可以是字符串或一个在每次匹配时调用的函数。
  • 如果 pattern 是字符串，则只会替换第一个匹配项。原始的字符串不会改变。
  • replacement是函数，将为每个匹配调用该函数，并将其返回值用作替换文本。

关于JS原生表单

表单事件

input事件，change事件

  • input:随着每一次的输入都会触发这个事件。连续触发
  • change:跟input事件很像，不同点在于是在光标失去焦点的时候触发
    这两个事件一般用在input和textarea上时有区分

focus事件，blur事件

  • focus:元素节点获得焦点的时候触发，从失去焦点到获得焦点才触发
  • blur:元素节点失去焦点的时候触发，从聚焦到失去焦点
    以上两个事件不会冒泡

addEventListener第三个参数默认是false，指在事件冒泡阶段触发事件处理函数

submit事件，reset事件

  • submit 事件：当表单数据向服务器提交的时候触发(发生在form上)
  • reset 事件：当表单重置的时候触发(发生在form上)
  • invalid 事件：提交的表单里面的数据如果有不满足条件的就会触发

前端画圆的方式

vuex存储数据存在哪里？浏览器刷新后数据会不会丢失？
Vuex 的默认数据是存储在内存中的。也就是说：
数据保存在浏览器中页面加载后创建的 JavaScript 对象中；数据存在于运行时内存里，而不是写入本地文件或浏览器存储（如 localStorage 或 sessionStorage）中。
刷新页面后，Vue 实例和 Vuex store 会被重新创建，而原来的内存数据就会被清除，所以刷新后数据会丢失

如果你是一个普通用户，你有A,B,C三个页面的访问权限，然后你现在登录到系统里面，在B页面中进行操作，同时系统管理员在你使用B页面期间把你的权限进行了更改，你此时不再拥有B页面的访问权限，作为一个前端，你该如何处理这种情况?由于缓存的缘故你仍旧在b页面，如何重新进行一个判断？如何刷新？在什么时机触发？

数组与类数组

特性 数组 类数组
类型 Array 的实例（arr instanceof Array === true） 普通对象（如 arguments、NodeList、HTMLCollection）

Symbol.iterator 默认实现，可迭代 无迭代器（除非手动实现）

JavaScript 中的变量提升和函数提升
函数声明（function getName() {}）会被提升到作用域顶部，并且赋值也提前。
变量声明（var getName）也会被提升，但赋值不会提前。

new Foo().getName(); // 3
new Foo() 创建了一个实例对象
实例对象的原型是 Foo.prototype
所以 new Foo().getName() 会去原型链上找方法
找到 Foo.prototype.getName，输出 3

new new Foo().getName(); // 3

new Foo() 创建实例对象
new Foo().getName是原型方法 function () { alert(3); }
所以 new new Foo().getName() 相当于 new (new Foo().getName)()
也就是调用 getName 构造函数，输出 3
