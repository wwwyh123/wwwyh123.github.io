String.prototype.replace()

replace(pattern,replacement) 方法返回一个新字符串，其中一个、多个或所有匹配的 pattern 被替换为 replacement。

  • pattern 可以是字符串或 RegExp，replacement 可以是字符串或一个在每次匹配时调用的函数。
  • 如果 pattern 是字符串，则只会替换第一个匹配项。原始的字符串不会改变。
  • replacement是函数，将为每个匹配调用该函数，并将其返回值用作替换文本。

关于JS原生表单

表单事件

input事件，change事件

  • input:随着每一次的输入都会触发这个事件。连续触发
  • change:跟input事件很像，不同点在于是在光标失去焦点的时候触发
    这两个事件一般用在input和textarea上时有区分

focus事件，blur事件

  • focus:元素节点获得焦点的时候触发，从失去焦点到获得焦点才触发
  • blur:元素节点失去焦点的时候触发，从聚焦到失去焦点
    以上两个事件不会冒泡

addEventListener第三个参数默认是false，指在事件冒泡阶段触发事件处理函数

submit事件，reset事件

  • submit 事件：当表单数据向服务器提交的时候触发(发生在form上)
  • reset 事件：当表单重置的时候触发(发生在form上)
  • invalid 事件：提交的表单里面的数据如果有不满足条件的就会触发

数据保存在浏览器中页面加载后创建的 JavaScript 对象中；数据存在于运行时内存里，而不是写入本地文件或浏览器存储（如 localStorage 或 sessionStorage）中。

setTimeout(1000) 一定是 1000ms 吗？（定时不准原因）

  1. 主线程阻塞（最常见原因）
    如果主线程正在执行耗时任务（如大量计算、复杂 DOM 操作），即使 setTimeout 的时间到了，回调也必须排队等待主线程空闲。
  2. 最小延迟限制（浏览器/系统限制）
    浏览器对 setTimeout 有最小延迟限制，通常为 4ms（即使你写 setTimeout(fn, 0)）。
    在嵌套调用或低优先级任务中，可能被延长到 10ms 或更多。
  3. 其他宏任务竞争
    JavaScript 的事件循环中，setTimeout 回调属于宏任务（macrotask），它需要和其他宏任务（如 setInterval、I/O、DOM事件）竞争执行时机。

事件循环：

浏览器中的 JavaScript 只有一个主线程，负责执行代码、处理用户交互和渲染页面。如果直接在主线程执行耗时操作（如定时器、网络请求），会导致页面“卡顿”或“阻塞”。事件循环（Event Loop）的作用：通过 异步任务调度 和 任务队列管理，让 JavaScript 在单线程环境下高效处理并发任务（如 I/O 操作、定时器、用户事件等），避免阻塞主线程。

  1. 同步任务优先：所有同步代码会立即执行，阻塞后续代码直到调用栈清空。
  2. 微任务优先级更高：微任务队列会在每次宏任务结束后立即清空，优先于下一个宏任务。
  3. 异步任务分类：
  • 宏任务：setTimeout、setInterval、I/O、UI 渲染等。
  • 微任务：Promise.then、MutationObserver、queueMicrotask 等。

Vue 是一个用于构建用户界面的 渐进式 JavaScript 框架，由尤雨溪开发，它的设计哲学是‘易用、灵活、高效’。”

“所谓‘渐进式’，意味着你可以从一个简单的 CDN 引入开始，只用它的模板语法做数据绑定；也可以逐步引入 Vue Router、Vuex/Pinia、Vite 等生态工具，构建大型单页应用（SPA）。这种灵活性让它非常适合从小项目到企业级应用的过渡。”

Vue 的核心特性包括：

  • 响应式系统：基于 Proxy（Vue3）实现数据劫持，数据变化自动更新视图，开发者无需手动操作 DOM。
  • 组件化开发：通过 .vue 单文件组件（SFC），把模板、逻辑、样式封装在一起，提升复用性和可维护性。
  • 虚拟 DOM：通过 render 函数生成 VNode，结合 diff 算法高效更新真实 DOM，提升性能。
  • 组合式 API（Composition API）：在 Vue3 中引入，相比选项式 API，它让逻辑复用更灵活（比如用 useMouse、useFetch 等自定义 Hook），特别适合复杂组件的逻辑拆分。
    丰富的生态系统：Vue Router 做路由，Pinia 做状态管理（推荐替代 Vuex），Vite 做极速构建，还有 Nuxt.js 做服务端渲染。”
    “我在项目中使用 Vue3 + Composition API + TypeScript + Vite 的技术栈，比如做过一个后台管理系统，通过 ref、reactive 管理状态，用 onMounted 发起请求，computed 处理表格数据，watch 监听搜索条件变化。还用 defineComponent 和 Prop 类型定义组件接口，提升了代码可读性和类型安全。”
