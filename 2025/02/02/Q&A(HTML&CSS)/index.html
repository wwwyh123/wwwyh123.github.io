html5新特性

  1. 语义化标签：<header>,<footer>,<article>,<section>,<nav>
  2. 多媒体支持：<audio>,<video>
  3. 表单增强：新增了多种输入类型：如<input type="email">,date,range等；属性如placeholder,<input type="text" name="usrname" required>；提升了表单功能和用户体验
  4. Canvas和SVG

html元素分类

行内元素

<span>,<a>,<strong>,<b>、<em>,<i>、<del>、<s>、<ins>、<u>、<img/>、<input/>、<select>、<textarea>、<br/>
行内元素的特点：
① 相邻行内元素在一行上，一行可以显示多个。
② 高、宽直接设置是无效的。
③ 默认宽度就是它本身内容的宽度。
④ 行内元素只能容纳文本或其他行内元素。
注意：
 链接里面不能再放链接
 特殊情况链接 < a > 里面可以放块级元素，但是给 < a > 转换一下块级模式最安全

行内块元素

在行内元素中有几个特殊的标签 —— < img />、< input />、< td >，它们同时具有块元素和行内元素的特点。 有些资料称它们为行内块元素。
行内块元素的特点：
① 和相邻行内元素（行内块）在一行上，但是他们之间会有空白缝隙。一行可以显示多个（行内元素特点）。
② 默认宽度就是它本身内容的宽度（行内元素特点）。
③ 高度，行高、外边距以及内边距都可以控制（块级元素特点）

块级元素

div、p、h1~h6、ul、ol、dl、li、dd、table、hr、blockquote、address、table、menu、pre，HTML5新增的header、section、aside、footer等

<iframe>：内嵌框架元素

它能够将另一个 HTML 页面嵌入到当前页面中。

id 属性
唯一标识：id 属性为 <iframe> 元素提供了一个唯一的标识符，在整个HTML文档中应该是唯一的。
DOM操作：通过 id，你可以方便地使用 JavaScript 或 CSS 选择器来访问和操作该 <iframe> 元素。例如，使用 document.getElementById('yourIframeId') 来获取该 iframe 的 DOM 对象。
目标框架名称：name 属性主要用于指定 iframe 的名称，它通常与超链接或表单的目标属性一起使用，以确定内容应在哪个 iframe 中显示。
表单提交：当使用

标签时，可以通过设置 target 属性为某个 iframe 的 name 值，使得表单提交的结果显示在该 iframe 中。
示例：
html
深色版本
<iframe name="targetFrame" src="https://example.com"></iframe>
<a href="https://another-example.com" target="targetFrame">Go to Another Example</a>
在这个例子中，点击链接会将页面加载到名为 targetFrame 的 iframe 中。

flex

Flexible Box简称flex，意为”弹性布局”，可以简便、完整、响应式地实现各种页面布局
采用Flex布局的元素，称为flex容器container
它的所有子元素自动成为容器成员，称为flex项目item

容器属性

flex-direction

决定主轴方向

  • row(默认值)：主轴为水平方向，起点在左端
  • row-reverse:主轴为水平方向，起点在右端
  • column:主轴方向为垂直方向，起点在上沿
  • column-reverse:主轴为垂直方向，起点在下沿

flex-wrap

  • nowrap(默认值)：不换行
  • wrap:换行
  • wrap-reverse:换行，第一行在上方

flex-flow

是flex-direction属性和flex-wrap属性的简写形式，默认值为row nowrap

justify-content

主轴对齐方式

  • flex-start(默认值):左对齐
  • flex-end:右对齐
  • center:居中
  • space-between:两端对齐(不留空隙)，项目之间的间隔都相等
  • space-around:两个item两侧间隔相等

align-items

定义item在交叉轴上如何对齐

  • flex-start : 交叉轴的起点对齐
  • flex-end:交叉轴的终点对齐
  • center:交叉轴的中点对齐
  • baseline:item的第一行文字的基线对齐
  • stretch(默认值):如果item未设置高度或设为auto，将占满整个容器的高度

align-content

定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用

  • flex-start：与交叉轴的起点对齐
  • flex-end：与交叉轴的终点对齐
  • center：与交叉轴的中点对齐
  • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布
  • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍
  • stretch（默认值）：轴线占满整个交叉轴

容器成员属性

  • order：定义项目的排列顺序。数值越小，排列越靠前，默认为0
  • flex-grow:上面讲到当容器设为flex-wrap: nowrap;不换行的时候，容器宽度有不够分的情况，弹性元素会根据flex-grow来决定定义项目的放大比例（容器宽度>元素总宽度时如何伸展）默认为0，即如果存在剩余空间，也不放大
  • flex-shrink
  • flex-basis
  • flex：flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto
  • align-self

css中隐藏元素的方法

display:none;

元素本身占有的空间就会被其他元素占有，也就是说它会导致浏览器的重排和重绘
消失后，自身绑定的事件不会触发，也不会有过渡效果
特点：元素不可见，不占据空间，无法响应点击事件

opacity:0;

由于其仍然是存在于页面上的，所以他自身的的事件仍然是可以触发的，但被他遮挡的元素是不能触发其事件的
需要注意的是：其子元素不能设置opacity来达到显示的效果
特点：改变元素透明度，元素不可见，占据页面空间，可以响应点击事件

visibility:hidden;

给人的效果是隐藏了，所以他自身的事件不会触发
特点：元素不可见，占据页面空间，无法响应点击事件

position:absolute 将元素移除可视区域

display:none;与visibility:hidden;

  • 共同点：都可以让元素不可见
  • 不同点：1. display:none;会让元素从渲染树中完全消失，不渲染时不占据任何空间。而visibility:hidden不会让元素从渲染树中消失，不渲染时元素继续占据空间，只是内容不可见；
  • display是非继承属性 visibility是继承属性

为什么要初始化CSS样式

因为浏览器有兼容问题，不同浏览器对有些元素的默认样式是不同的，并且默认的样式会影响到项目最终的布局，如果没有CSS初始化，那么有可能会出此浏览器之间的显示页面差异

盒模型是什么？标准盒模型和怪异盒模型的区别？

盒模型是描述元素在页面中占据空间的模型，由4部分组成：

  • 内容区(content)
  • 内边距(padding)
  • 边框(border)
  • 外边距(margin)
    标准盒模型（content-box）：元素总宽度 = width + padding + border
    怪异盒模型（border-box）：元素总宽度 = width（包含padding和border）
    👉 通过box-sizing: border-box切换

Flex布局中justify-content和align-items的区别是什么？

justify-content控制主轴（默认水平方向）的排列方式，align-items控制交叉轴（默认垂直方向）的排列方式。
👉 补充：如果父元素设置了flex-direction: column，主轴会变成垂直方向！

  2. 内联样式（style=”…”）(1000)

如何用CSS实现一个三角形？

透明边框 的作用是占据空间并形成斜角，迫使目标边框收缩为三角形。
三角形底边长度 = 左右（或上下）边框宽度之和
三角形高度 = 目标边框的宽度

什么是BFC（块级格式化上下文）？如何触发BFC？列举应用场景。

https://juejin.cn/post/7031065166317879310

  • BFC 是一个独立渲染的区域，内部元素的布局不会影响外部元素
  • 触发条件
    • overflow:hidden / auto / scroll
    • display: inline-block / table-cell / flex / grid
    • position: absolute / fixed
    • float:left / right
  • 应用条件：
    • 避免外边距合并： 两个相邻元素的上下外边距会合并，BFC 可隔离他们
    • 清除浮动：父元素触发BFC可包含浮动子元素
    • 阻止元素被浮动覆盖：非浮动元素触发BFC后，不会与浮动元素重叠

BFC解决外边距合并问题的原理

父元素创建BFC后，内部子元素的外边距被限制在父容器内，不会与外部元素的外边距重叠。

CSS中position: sticky的作用是什么？和fixed有何区别？

sticky的作用:元素根据滚动位置在相对定位（relative）和固定定位（fixed）之间切换。
与 fixed 的区别：

  • fixed：直接相对于视口定位，不保留原始空间。
  • sticky：在父容器内滚动到阈值（如 top: 10px）后变为固定定位，保留原始空间。

如何实现一个自适应正方形（宽高相等）的盒子？

  • 利用padding百分比（相对于父元素宽度）

    padding-top: 50%
    这条规则设置元素的内边距（padding）。padding-top 的值是 50%，但它的 50% 是相对于元素的宽度来计算的。
    padding的计算是基于父元素的宽度
    这意味着 .square 元素的上边距会等于它的宽度的 50%（即宽度的一半）。
    由于内边距是在元素的内容区之上，因此它增加了元素的高度。如果元素宽度是 50%（父容器的 50%），则内边距 padding-top 会让元素的高度变成宽度的 50%，从而使元素的高度等于宽度。

  • 使用vw单位

CSS选择器div > p和div p的区别是什么？

div > p：选择 div 的直接子元素

（仅一代）。

div p：选择 div 的所有后代元素 <p>（不限层级）。
在 CSS 中，选择 <div> 中的第一个 <p> 元素有多种方法

  • first-child ：只有当p是div的第一个子元素时才会生效
  • first-of-type：即使p不是第一个子元素，只要是第一个p类型元素就会生效
  • nth-child(1): 只有当p是div的第一个子元素时才会生效
  • nth-of-type(1):即使p不是第一个子元素，只要是第一个p类型元素就会生效
    在js中const firstP = document.querySelector('div p');

cookies,sessionStorage和localStorage的区别

  • cookies是网站用来标记用户身份的一段数据，通常情况下是一段加密的字符串，并且默认情况下**只会在同源的HTTP(同域名同端口同协议)**请求中携带
  • sessionStorage是浏览器本地存储的一种方式，以键值对的形式进行存储，并且存储的数据会在浏览器关闭之后自动删除
  • localStorage是浏览器本地存储的一种方式，和sessionStorage一样，也是以键值对的形式存在，不过它存储的是一个持久化的数据，一般不主动删除，数据会一直存在

实现渐变色边框

  1. 使用border-image和linear-gradient background定义了两个背景层：
  • 第一层背景：
    linear-gradient(white, white) padding-box
    这是一个纯白色（从白色到白色的线性渐变）的背景图层，应用在 padding-box 区域。padding-box 指的是包含内边距（padding）的区域，但不包括边框（border）。
  • 第二层背景：
    linear-gradient(to right, red, orange, yellow, green, blue, indigo, violet) border-box
    这是一个从红色到紫色的多色线性渐变背景图层，应用在 border-box 区域。border-box 包含了边框、内边距和内容区域。
    border-image: source slice width outset repeat|initial|inherit;
    slice是从四个角向内切割
    border-image-repeat 用于设置图像边界是否应重复（repeat）、拉伸（stretch）或铺满（round）。
