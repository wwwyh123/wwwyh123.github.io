盒模型是什么？标准盒模型和怪异盒模型的区别？

盒模型是描述元素在页面中占据空间的模型，由4部分组成：

  • 内容区(content)
  • 内边距(padding)
  • 边框(border)
  • 外边距(margin)
    标准盒模型（content-box）：元素总宽度 = width + padding + border
    怪异盒模型（border-box）：元素总宽度 = width（包含padding和border）
    👉 通过box-sizing: border-box切换

  • 方法2：Grid布局
  • 方法3：定位 + transform
  • 方法四：表格布局

flex:1

是flex-grow: 1、flex-shrink: 1、flex-basis: 0% 的简写，表示该元素会占据剩余空间。

Flex布局中justify-content和align-items的区别是什么？

justify-content控制主轴（默认水平方向）的排列方式，align-items控制交叉轴（默认垂直方向）的排列方式。
👉 补充：如果父元素设置了flex-direction: column，主轴会变成垂直方向！

什么是BFC（块级格式化上下文）？如何触发BFC？列举应用场景。

https://juejin.cn/post/7031065166317879310

  • BFC 是一个独立渲染的区域，内部元素的布局不会影响外部元素
  • 触发条件
    • overflow:hidden / auto / scroll
    • display: inline-block / table-cell / flex
    • position: absolute/fixed
    • float:left /right
  • 应用条件：
    • 避免外边距合并： 两个相邻元素的上下外边距会合并，BFC 可隔离他们
    • 清除浮动：父元素触发BFC可包含浮动子元素
    • 阻止元素被浮动覆盖：非浮动元素触发BFC后，不会与浮动元素重叠

CSS中position: sticky的作用是什么？和fixed有何区别？

sticky的作用:元素根据滚动位置在相对定位（relative）和固定定位（fixed）之间切换。
与 fixed 的区别：

  • fixed：直接相对于视口定位，不保留原始空间。
  • sticky：在父容器内滚动到阈值（如 top: 10px）后变为固定定位，保留原始空间。

如何实现一个自适应正方形（宽高相等）的盒子？

  • 利用padding百分比（相对于父元素宽度）

    padding-top: 50%
    这条规则设置元素的内边距（padding）。padding-top 的值是 50%，但它的 50% 是相对于元素的宽度来计算的。
    padding的计算是基于父元素的宽度
    这意味着 .square 元素的上边距会等于它的宽度的 50%（即宽度的一半）。
    由于内边距是在元素的内容区之上，因此它增加了元素的高度。如果元素宽度是 50%（父容器的 50%），则内边距 padding-top 会让元素的高度变成宽度的 50%，从而使元素的高度等于宽度。

  • 使用vw单位

let、const、var的区别是什么？为什么要用let取代var？

变量提升：var 声明的变量会提升到作用域顶部，初始值为undefined。

  • let 块级作用域，可重新赋值，但不可重复声明
  • const 块级作用域，声明时必须初始化，不可重新赋值，但对象/数组的内容可修改（引用不变）。
  • var 函数作用域，存在变量提升（声明被提升到作用域顶部，但赋值不提升）,允许重复说明
  1. 更严格的作用域控制：避免变量泄漏，提升代码可预测性。
  2. 减少隐藏错误：通过暂时性死区和禁止重复声明，提前暴露问题。

什么是闭包？举一个实际应用场景的例子。

函数嵌套时，内层函数引用了外层函数的变量，外层函数执行完毕后，这些变量依然被内层函数持有，形成闭包。

  • 防抖（Debounce）：限制函数在短时间内频繁触发（如搜索框输入）。
  • 节流（Throttle）：固定时间间隔执行函数（如滚动事件）。

Promise.all和Promise.allSettled的区别是什么？

Promise.all：所有 Promise 都成功时，返回一个包含所有结果的数组。如果有一个 Promise 失败，立即返回失败的原因。
Promise.allSettled：所有 Promise 都完成后（无论成功或失败），返回一个包含所有结果的数组。

for in 和 for of 的区别

  1. 迭代内容不同：for-in 循环迭代对象的键名，而 for-of 循环迭代对象的值。
  2. 适用对象类型不同：for-in 循环适用于遍历对象，包括普通对象、数组、函数等；而 for-of 循环适用于遍历可迭代对象，包括数组、字符串、Map、Set 等。
  3. 迭代顺序不同：for-in 循环的迭代顺序是不确定的，而 for-of 循环的迭代顺序是确定的。
  4. 原理不同：for-in 循环会遍历对象的原型链，并包含从原型链继承的属性；而 for-of 循环遍历的对象是可迭代对象，其值是可枚举的

Vue的生命周期钩子中，created和mounted的区别是什么？

  • created：实例已创建，但DOM未挂载（不可操作DOM）。
  • mounted：DOM已挂载到页面，可操作DOM。
    适用场景：
  • created：初始化数据（如API请求）。
  • mounted：操作DOM或集成第三方库（如初始化图表）。

什么是虚拟DOM？为什么需要它？

虚拟DOM（Virtual DOM） 是一个轻量级的JavaScript对象，描述真实DOM的结构。
作用：
性能优化：通过Diff算法对比新旧虚拟DOM，最小化真实DOM操作。
跨平台：抽象渲染层（如渲染到Web、Native）。
Vue中的虚拟DOM：通过render函数生成，更新时对比差异后局部更新真实DOM。

数据代理

通过vm对象来代理data对象中的属性的操作。vm身上有的是_data，而没有data。然后通过Object.definePropoty的getter读取到_data里的数据复制一份到vm身上。
好处：更加方便的操作data中的数据
基本原理：通过Object.defineProperty()把data对象中的所有属性添加到vm上。为每一个添加到vm上的属性都指定一个getter和setter。在getter/setter内部去操作（读/写）data中对应的属性。

Vue2和Vue3在响应式原理上的主要区别是什么？

Vue2:使用Object.defineProperty()监听数据变化，访问时触发getter，修改时触发setter来更新视图，但它有局限性：无法监听数组索引的内容更改和新属性的添加与删除。需要借助(Vue.set())
Vue3:采用了 Proxy 来实现响应式，它能劫持整个对象及其嵌套的对象，能够监听数组索引、对象的新增属性等，性能和灵活性更好。

Vue中的nextTick是什么？它的作用是什么？

Vue 提供的一个 API，用于在 DOM 更新后执行某些操作。Vue 会将 DOM 更新操作和 JavaScript 执行操作放入异步队列中。
使用场景： 1. 修改数据后立即操作DOM 2. 确保视图更新后再执行某些逻辑

Vue的v-if和v-show有什么区别？性能上有何差异？

v-if：根据条件动态地创建和销毁 DOM 元素。当条件为 false 时，元素会被销毁；当条件为 true 时，元素会被创建。适用于条件变化不频繁的场景。
v-show：通过切换元素的 display 样式来控制元素显示与否，元素会始终存在 DOM 中，只是改变其 CSS 样式的显示或隐藏。适用于频繁切换的场景。

什么是Vue的混入（Mixin）？它的优缺点是什么？

Mixin 是 Vue 提供的一种机制，它允许将多个组件的逻辑提取并共享到不同的组件中。通过定义一个包含 data、methods、computed，生命周期钩子 等的混入对象，在多个组件中复用该逻辑。
优点：代码复用性高，能够避免逻辑的重复。
缺点：可能导致命名冲突（尤其是当多个 mixin 中有相同的方法、数据时），会增加调试和维护的复杂度。大规模使用时，可能让代码变得难以追踪。

Vue3的<script setup>语法糖解决了什么问题？

<script setup>是 Vue3 提供的一种新语法糖，用于简化组件的编写。它通过<script setup>标签，减少了重复的 import 和 return，并且优化了组件的编译过程，提升了性能。它主要解决了 Composition API 使用时的样板代码问题，使代码更加简洁和易读。

  • 更少的样板代码：不需要 export default 或 setup() 函数。
  • 直接使用组合式 API：可以直接使用 ref, computed 等 Vue 3 的组合式 API。
  • 模板访问：在 <script setup> 中声明的所有变量和函数都可以直接在模板中访问。
  • 类型支持：支持 TypeScript，可以直接声明 props 和自定义事件的类型
  • 性能优化：由于模板会被编译成同一作用域内的渲染函数，避免了渲染上下文代理对象，从而提高了运行时性能

浏览器与网络

从输入URL到页面渲染的完整过程是什么？

  1. DNS解析：浏览器解析 URL 中的域名，查询其对应的 IP 地址。
  2. TCP连接：浏览器与服务器建立 TCP 连接（通常使用三次握手）。
  3. 发送 HTTP 请求：浏览器向服务器发送 HTTP 请求。
  4. 服务器处理请求并响应：服务器根据请求返回 HTML、CSS、JS 等资源。
  5. 浏览器渲染页面：浏览器开始解析 HTML 内容，构建 DOM 树，解析 CSS，生成 CSSOM 树，然后合并生成渲染树，最后绘制页面。

什么是跨域？CORS的原理是什么？如何解决跨域问题？

指在浏览器中，不同源的请求（不同的域名、端口或协议）会受到浏览器的安全限制。比如，前端向不同域的 API 发送请求会被浏览器拦截。
CORS（跨域资源共享）：是一种浏览器机制，允许服务器明确指定哪些源可以访问资源。通过 HTTP 头信息（如 Access-Control-Allow-Origin）告知浏览器，哪些请求可以跨域访问。
