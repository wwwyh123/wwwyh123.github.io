  • 两个都为简单类型，字符串和布尔值都会转换成数值，再比较
  • 简单类型与引用类型比较，对象转化成其原始类型的值，再比较

let、const、var的区别是什么？为什么要用let取代var？

变量提升：var 声明的变量会提升到作用域顶部，初始值为undefined。

  • let 块级作用域，可重新赋值，但不可重复声明
  • const 块级作用域，声明时必须初始化，不可重新赋值，但对象/数组的内容可修改（引用不变）。
  • var 函数作用域，存在变量提升（声明被提升到作用域顶部，但赋值不提升）,允许重复说明
  1. 更严格的作用域控制：避免变量泄漏，提升代码可预测性。
  2. 减少隐藏错误：通过暂时性死区和禁止重复声明，提前暴露问题。

什么是闭包？举一个实际应用场景的例子。

函数嵌套时，内层函数引用了外层函数的变量，外层函数执行完毕后，这些变量依然被内层函数持有，形成闭包。
在一个函数环境中，闭包 = 函数 + 词法环境
产生内存泄漏的情况：

  1. 持有本该销毁的函数，造成其关联的词法环境无法销毁(移除一个监听事件时)
  2. 当有多个函数共享词法环境时，可能导致词法环境膨胀，从而发生无法访问但无法销毁的数据
  • 防抖（Debounce）：限制函数在短时间内频繁触发（如搜索框输入）。
  • 节流（Throttle）：固定时间间隔执行函数（如滚动事件）。
  • 函数柯里化
  • 数据封装：可以创建私有变量，避免全局污染
    闭包也存在着一个潜在的问题，由于闭包会引用外部函数的变量，但是这些变量在外部函数执行完毕后没有被释放，那么这些变量会一直存在于内存中。 一旦形成闭包，只有在页面关闭后，闭包占用的内存才会被回收，就会造成的内存泄漏。

因此我们在使用闭包时需要特别注意内存泄漏的问题，可以用以下两种方法解决内存泄露问题：

  1. 及时释放闭包：手动调用闭包函数，并将其返回值赋值为null，这样可以让闭包中的变量及时被垃圾回收器回收。
  2. 使用立即执行函数（IIFE，Immediately Invoked Function Expression）：在创建闭包时，将需要保留的变量传递给一个立即执行函数，并将这些变量作为参数传递给闭包函数，这样可以保留所需的变量，而不会导致其他变量的内存泄漏。

Promise.all和Promise.allSettled的区别是什么？

Promise.all：所有 Promise 都成功时，返回一个包含所有结果的数组。如果有一个 Promise 失败，立即返回失败的原因。
Promise.allSettled：所有 Promise 都完成后（无论成功或失败），返回一个包含所有结果的数组。

解释async/await的底层实现原理（结合生成器Generator）。

事件循环 EventLoop

JavaScript 是单线程的编程语言，(一次只能执行一个任务)。然而，为了处理异步操作（如网络请求、定时器等），JavaScript 引入了事件循环机制来管理这些异步任务的执行顺序。

同步任务指：在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务

异步任务：不进入主线程，而进入“任务队列”的任务，只有“任务队列”通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。

而当主线程中的任务完成之后，主线程从任务队列中读取事件，这个过程是循环不断的。

async/await

ES8引入的语法糖；用于简化promise的使用。async函数返回一个promise;await用于等待promise的解决
JavaScript 是单线程的，但它通过事件循环和任务队列实现了非阻塞的异步操作。异步操作完成后，它们的结果会被放入微任务队列中，以便在当前任务完成后立即执行。

async 和 await 的实现依赖于 JavaScript 的事件循环机制。具体来说：

Promise：async 函数内部的操作通常涉及 Promise 对象。Promise 提供了一种标准化的方式来处理异步操作的结果或错误。
微任务：当一个 Promise 被解决时，它的回调函数会被放入微任务队列中，并在当前任务完成后立即执行。这使得 await 能够在 Promise 解决后恢复函数的执行。

如果 JavaScript 是单线程的，它是如何实现异步编程的？

JavaScript 利用浏览器的多线程能力 + 事件循环机制
虽然 JavaScript 是单线程语言，但它通过 事件循环（Event Loop）机制、回调函数、消息队列 和 浏览器提供的异步 API 来实现异步编程。

  1. JavaScript 是单线程的
  • JS 引擎（如 V8）只有一个主线程来执行代码。
  • 这意味着一次只能做一件事，不能并行执行多个任务。
  • 如果所有操作都同步执行，遇到耗时任务（如网络请求、定时器），页面就会“卡死”。
  1. 浏览器是多线程的
    虽然 JS 是单线程的，但浏览器不是。浏览器有多个线程来处理不同的任务：
浏览器线程 作用
JS 引擎线程 执行 JavaScript 代码
渲染线程 渲染页面
定时器线程 处理 setTimeout / setInterval
网络线程 发起网络请求（如 AJAX）
事件触发线程 监听用户交互事件（如 click、keydown）

这些线程可以与 JS 主线程通信，从而实现异步。

for in 和 for of 的区别

  1. 迭代内容不同：for-in 循环迭代对象的键名，而 for-of 循环迭代对象的值。
  2. 适用对象类型不同：for-in 循环适用于遍历对象，包括普通对象、数组、函数等；而 for-of 循环适用于遍历可迭代对象，包括数组、字符串、Map、Set 等。
  3. 迭代顺序不同：for-in 循环的迭代顺序是不确定的，而 for-of 循环的迭代顺序是确定的。
  4. 原理不同：for-in 循环会遍历对象的原型链，并包含从原型链继承的属性；而 for-of 循环遍历的对象是可迭代对象，其值是可枚举的

为什么 0.1 + 0.2 !== 0.3？

  1. JavaScript 中的数字是以二进制形式存储的。0.1和0.2 在二进制中是无限循环小数，无法精确表示
  2. 精度丢失：由于js使用64位双精度浮点数，只能存储有限的位数；存储和计算的过程中，0.1和0.2的二进制表示会被截断，导致精度丢失
  3. 计算结果是一个接近0.3但不完全等于0.3的值
    如何解决浮点数精度问题
  4. 使用toFixed()将结果保留指定的小数位数；注意toFixed()返回的是字符串，需要转换为数字
  5. 使用整数运算，最后再转回小数

  1. 全局变量：在浏览器环境中，全局变量（包括挂载在window 对象上的属性）不会被垃圾回收机制自动回收，除非显式地将其删除或重新赋值。
  2. 未解除的事件监听器:当你为DOM元素添加事件监听器时，如果这些元素被移除或替换，但事件监听器仍然存在，它们将保持对这些元素的引用，导致内存泄漏。button.removeEventListener('click', handleClick);
  3. 定时器和回调函数:使用 setInterval 或 setTimeout 设置的定时器，如果定时器没有被清除，回调函数中的引用会阻止相关对象被垃圾回收。

defer 和 async 的区别

js的三种加载方式

  1. 正常模式<script src="script.js"></script>读到就加载并执行
  2. async模式<script async src="script.js"></script>;script.js会被异步加载，即加载和渲染后续文档元素的过程将和 script.js 的加载并行进行（异步）。当 script.js加载完整立即执行script.js。执行script.js时，**html解析暂停**。从加载完成立即执行来看，async模式 执行顺序与写的顺序无关，不保证执行顺序。
  3. defer模式<script defer src="index.js"></script>
    有 defer，script.js会被异步加载，即加载和渲染后续文档元素的过程将和 script.js 的加载并行进行（异步）。这一点与async模式一致。
    不同的是当 script.js加载完成并不会立即执行，而是在所有元素解析完成之后，DOMContentLoaded 事件触发之前完成。因此它会按照写的顺序执行。
    defer 属性
  • 执行时机：带有 defer 属性的脚本会在文档解析完成后，但在 DOMContentLoaded 事件触发之前执行。
  • 执行顺序：多个带有 defer 的脚本会按照它们在文档中出现的顺序依次执行。
  • 加载行为：脚本会被异步下载，但不会立即执行，直到文档解析完成。
  • 适用场景：适用于需要依赖 DOM 完全加载后才能正确运行的脚本。
    async 属性
  • 执行时机：带有 async 属性的脚本一旦下载完成就会立即执行，这可能会在文档完全解析之前发生。
  • 执行顺序：多个带有 async 的脚本不保证按顺序执行；每个脚本在其下载完成后立即执行。
  • 加载行为：脚本被异步下载，并且一旦下载完成就会立即执行，无论文档是否已经解析完毕。
  • 适用场景：适用于独立于其他脚本以及页面内容的脚本，例如分析或广告脚本。

垃圾回收机制

  1. 可达性：一个对象如果可以从根对象通过引用链访问到，则认为它是可达的。根对象通常包括全局变量、活动线程栈中的局部变量等。
  2. 不可达对象：无法从根对象访问到的对象被认为是不可达的，可以被垃圾回收器回收。
    常见的垃圾回收算法
  3. 引用计数法：每个对象都有一个引用计数器，记录有多少个引用指向该对象。当有新的引用指向该对象时，引用计数加一；当引用解除时，引用计数减一。当引用计数变为零时，表示没有引用指向该对象，可以立即回收其占用的内存
  4. 标记清除法：(标记阶段)从根对象开始遍历所有可达对象，并标记它们为可达；(清除阶段)遍历整个堆内存，回收未被标记的对象。

document.addEventListener(‘DOMContentLoaded’, …)

用于监听文档加载状态的事件处理器。它确保你的代码在 DOM（文档对象模型）完全加载并解析之后执行，但不会等待样式表、图片和子框架完成加载。
作用：

  1. 确保DOM已加载，可以安全地访问和操作DOM元素
  2. 提高性能：与 window.onload 不同，DOMContentLoaded 不会等待所有资源（如图片、样式表等）加载完成。因此，使用 DOMContentLoaded 可以让你的脚本更快地执行，从而提升页面的响应速度。
