原始类型vs引用数据类型

原始数据类型包括：string、number、boolean、undefined、null、symbol、bigint。
存储方式：值直接存储在栈内存中。
引用类型：包括：Object、Array、Function、Date 等。
存储方式：值存储在堆内存中，栈中存储的是堆内存的地址引用。
typeof null; 返回 ‘object’（历史遗留问题）

类型监测的方法

  • typeof
  • instanceof
  • Object.prototype.toString.call()

let、const、var的区别是什么？为什么要用let取代var？

变量提升：var 声明的变量会提升到作用域顶部，初始值为undefined。

  • let 块级作用域，可重新赋值，但不可重复声明
  • const 块级作用域，声明时必须初始化，不可重新赋值，但对象/数组的内容可修改（引用不变）。
  • var 函数作用域，存在变量提升（声明被提升到作用域顶部，但赋值不提升）,允许重复说明
  1. 更严格的作用域控制：避免变量泄漏，提升代码可预测性。
  2. 减少隐藏错误：通过暂时性死区和禁止重复声明，提前暴露问题。

什么是闭包？举一个实际应用场景的例子。

函数嵌套时，内层函数引用了外层函数的变量，外层函数执行完毕后，这些变量依然被内层函数持有，形成闭包。

  • 防抖（Debounce）：限制函数在短时间内频繁触发（如搜索框输入）。
  • 节流（Throttle）：固定时间间隔执行函数（如滚动事件）。
  • 函数柯里化
  • 数据封装：可以创建私有变量，避免全局污染
    闭包也存在着一个潜在的问题，由于闭包会引用外部函数的变量，但是这些变量在外部函数执行完毕后没有被释放，那么这些变量会一直存在于内存中。 一旦形成闭包，只有在页面关闭后，闭包占用的内存才会被回收，就会造成的内存泄漏。

因此我们在使用闭包时需要特别注意内存泄漏的问题，可以用以下两种方法解决内存泄露问题：

  1. 及时释放闭包：手动调用闭包函数，并将其返回值赋值为null，这样可以让闭包中的变量及时被垃圾回收器回收。
  2. 使用立即执行函数（IIFE，Immediately Invoked Function Expression）：在创建闭包时，将需要保留的变量传递给一个立即执行函数，并将这些变量作为参数传递给闭包函数，这样可以保留所需的变量，而不会导致其他变量的内存泄漏。

箭头函数没有自己的this，super，arguments。

箭头函数没有prototype属性。

解释原型链（Prototype Chain）是什么？如何实现继承？

原型链是 JavaScript 实现继承的机制。每个对象都有一个内部属性（可以通过__proto__访问），指向其原型对象。原型对象也有自己的原型，形成一条链，直到null
继承：对象可以访问原型链上的属性和方法

原型与原型链

每个函数都有一个prototype属性，这个属性会指向一个对象，就是通过调用该构造函数而创建的实例的原型，可以通过实例对象的__proto__来访问到这个原型对象
实例在创建时就会默认关联原型，并且会从原型继承属性
而每一个原型对象又都会存在一个constructor属性，这个属性会指向关联的这个构造函数
原型链：
如果访问一个实例对象的属性时，如果说这个实例对象中没有这个属性，那么JS引擎就会去该实例对象的原型对象中去找；如果属性在原型对象中也找不到，那么就会去原型的原型中去找，一直找到最上层的原型，找到空为止

Promise.all和Promise.allSettled的区别是什么？

Promise.all：所有 Promise 都成功时，返回一个包含所有结果的数组。如果有一个 Promise 失败，立即返回失败的原因。
Promise.allSettled：所有 Promise 都完成后（无论成功或失败），返回一个包含所有结果的数组。

事件循环 EventLoop

js语言的一大特点是单线程(同一个时间只能做一件事)。
同步任务指：在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务
异步任务：不进入主线程，而进入“任务队列”的任务，只有“任务队列”通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行。
而当主线程中的任务完成之后，主线程从任务队列中读取事件，这个过程是循环不断的。

  • instanceof:它是用来判断一个对象是否是另一个对象的实例，注意它只能用来判断对象

for in 和 for of 的区别

  1. 迭代内容不同：for-in 循环迭代对象的键名，而 for-of 循环迭代对象的值。
  2. 适用对象类型不同：for-in 循环适用于遍历对象，包括普通对象、数组、函数等；而 for-of 循环适用于遍历可迭代对象，包括数组、字符串、Map、Set 等。
  3. 迭代顺序不同：for-in 循环的迭代顺序是不确定的，而 for-of 循环的迭代顺序是确定的。
  4. 原理不同：for-in 循环会遍历对象的原型链，并包含从原型链继承的属性；而 for-of 循环遍历的对象是可迭代对象，其值是可枚举的

事件与事件流

事件流的三个阶段

  • 事件捕获阶段
  • 处于目标阶段
  • 事件冒泡阶段
    事件模型
  • 原始事件模型(DOM0级)
  • 标准事件模型(DOM2级)
  • IE事件模型(基本不用)
    原始事件模型(L0)：
  • 同一个对象，后面注册的事件会覆盖前面注册的(同一个事件)
  • 直接使用null覆盖就可以实现事件的解绑:btn.onclick=null
  • 只支持冒泡，不支持捕获
    两种方式：
  • HTML代码中直接绑定
    <input type="button" onclick="fun()">
  • 通过js代码绑定
    var btn = document.getElementById('.btn').onclick = fun;
    标准事件模型(L2):
  • 语法：addEventListener(事件类型，事件处理函数，是否使用捕获)
  • 后面注册的事件不会覆盖前面注册的事件(同一个事件)
  • removeEventListener(事件类型，事件处理函数，获取捕获或者冒泡阶段)
  • 匿名函数无法解绑

三个过程：

  • 事件捕获阶段
  • 事件处理阶段
  • 事件冒泡阶段

js中的this是如何工作的

this在js中表示当前执行上下文的对象。他的值取决于函数的调用方式

  1. 全局上下文：全局作用域中，this指向全局对象(在浏览器中是window,在node.js中是global)
  2. 函数上下文：普通函数中，this指向全局对象(非严格模式)或undefined(严格模式)；对象方法中：this指向调用该方法的对象；在构造函数中,this指向新创建的对象
  3. 箭头函数：没有自己的this，继承外层的this
  4. 显示绑定：可以用call,apply或者bind

为什么 0.1 + 0.2 !== 0.3？

  1. JavaScript 中的数字是以二进制形式存储的。0.1和0.2 在二进制中是无限循环小数，无法精确表示
  2. 精度丢失：由于js使用64位双精度浮点数，只能存储有限的位数；存储和计算的过程中，0.1和0.2的二进制表示会被截断，导致精度丢失
  3. 计算结果是一个接近0.3但不完全等于0.3的值
    如何解决浮点数精度问题
  4. 使用toFixed()将结果保留指定的小数位数；注意toFixed()返回的是字符串，需要转换为数字
  5. 使用整数运算，最后再转回小数

== 和 === 的区别，== 进行类型转换的原理？

  • == 抽象等比较：检查两个值是否相等，但在比较前会尝试进行类型转换。
    当比较一个数字和一个字符串时，字符串会被转换为数字。
    当比较一个布尔值和非布尔值时，布尔值会被转换为数字（true变为1，false变为0），除非另一个操作数是字符串或对象。
    当比较一个对象和一个原始类型时，对象会通过调用其valueOf()方法或者toString()方法被转换为原始类型后再进行比较。
  • ===（严格相等比较）
    功能：不仅检查两个值是否相等，还会检查它们的数据类型是否相同。如果类型不同，直接返回false，不会进行任何类型的转换。
