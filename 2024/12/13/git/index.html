工作区、暂存区、版本区，三个区共同组成了一个 Git 仓库

基本操作流程

  1. 创建一个空文件夹（名称不要包含中文），随后双击打开文件夹。
  2. 在文件夹空白处右键启动命令行
  3. 创建一个 Git 仓库，git init(工作区)，可以创建一些文件，我们创建a.txt
  4. git add a.txt把工作区中【a.txt】添加到暂存区;git add -A可以把工作区中的所有文件，都添加到暂存区.
  5. 把暂存区a.txt添加到版本区，具体命令为：git commit -m 'xxx'

查看仓库状态

git status或git status -s

  • On branch master(当前所在的分支)
    nothing to commit, working tree clean(当前没有任何文件需要提交)

    刚创建一个文件,还在工作区，没有接受版本控制。
  • 红色的含义：操作停留在工作区
  • ？的含义：当前文件还没有接受版本控制
    git add已经存入了暂存区
  • 绿色的含义：操作已经存入暂存区
  • A 的含义：新增（单词：add）
    对文件进行修改
  • M的含义：修改（单词： modify）

Gitee

步骤

  • 维护好本地仓库（所有文件均接受版本控制
    git add xxxx
    git commit -m '注释'
  • 配置本地仓库关联远程仓库：
    git remote add origin https://xxxx/xxxx/xxxxx.git
    • 查看当前仓库关联远程仓库地址：git remote -v
    • 更改远程仓库的 URL：git remote set-url origin 新的仓库URL
    • 删除与远程仓库的关联：git remote remove origin
  • 将本地仓库的 master 分支推送到远程仓库
    git push -u origin master
  • 当本地仓库想获得远程仓库的最新修改时，可以使用git pull命令拉取远端代码。

克隆远程库

获取一个远程仓库的地址，随后执行克隆命令
git clone https://xxxx/xxxx/xxxx.git
使用–depth 1 参数可以克隆仓库的最近一次提交，这样可以减少克隆所需的时间和空间。
git clone --depth 1 https://xxxx/xxxx/xxxx.git

如何撤销已提交的代码更改？

  • git revert命令通过创建一个新的提交来撤销之前的更改，保持历史记录完整。
  • git reset命令，该命令允许将分支指针移动到先前的提交，但需要谨慎使用，因为它会改变版本历史。
    A -- B -- C (HEAD)
    如果你想撤销最近的一次提交 C，可以这样做：
    git reset --soft HEAD~1
    现在提交历史变成：
    A -- B (HEAD)
    但是你的工作目录和暂存区仍然包含提交 C 的更改

创建一个新仓库并将本地代码push到远程

创建仓库

上传本地代码

  1. 设置用户名以及邮箱
    git config --global user.name "123"
    git config --global user.email "667@qq.com"
  2. 设置SSH公钥
  3. 添加生成的SSH公钥
  4. 在需要提交代码的文件夹中启动一个新的git bash
  • git add .//提交所有代码到中转站
  • git commit -m "注释内容"提交所有代码到本地仓库
  • git push 提交本地仓库到远程

一些命令：

  1. git init初始化仓库
  2. git config -l 查看配置(user.email user.name)
  3. git status 查看状态：红色的没有被git管理；绿色的存储在版本库暂缓区
  4. git add . 添加到暂缓区
  5. git commit -m '注释内容'
  6. git diff 查看修改文件 git diff index.js查看某一个文件
  7. git log 查看修改历史 git reflog (简单形式)
  8. 回到上一个版本git reset --hard HEAD^ 回到指定版本git reset --hard 版本号

分支操作

默认主分支:master

  1. 查看本地分支git branch
  2. 新建分支git branch 分支名称
  3. 切换分支git checkout 分支名称
  4. 查看远程仓库分支 git branch -r
  5. 本地分支提交到远程仓库git push --set-upstream origin 分支名称
  6. 删除本地分支git branch -d 分支名称
  7. 删除远程仓库的分支git push origin --delete 分支名称
  8. git merge 分支名称如果当前是在master分支下写入：git merge dev 那么master和dev会进行合并；相当于master+=dev
    本地没有dev分支，但远程有，可以直接切换git checkout dev

多人合作开发以及提交冲突问题

  1. 克隆下载代码
  2. 配置git config user.name '用户名',git config user.email '邮箱'
  3. git add .
  4. git commit -m '注释内容'
  5. git push
    git push可能会报错；原因：远程仓库有人更新了代码，不能直接git push
    解决：我们必须先把远程仓库的代码往下拉，然后再push;git pull,git push
    如何解决合并冲突？
    合并冲突发生在合并分支时，当两个分支的同一部分产生了不同的更改时。解决合并冲突的一般步骤如下：
  • 使用git status命令查看冲突的文件。
  • 打开冲突文件，手动解决冲突并选择要保留的更改。
  • 使用git add命令将解决冲突后的文件标记为已解决。
  • 使用git commit命令提交解决冲突的结果。
    git clone 用于从远程仓库完整地克隆一个新的本地仓库。这是你第一次从远程仓库获取项目代码时使用的命令。
    git pull 用于更新现有的本地仓库，使其与远程仓库同步。它通常是在已经存在本地仓库的情况下用来获取最新的更改。
    git pull是下面两个操作的融合
    git fetch：
    这个命令会从远程仓库获取最新的更改，但不会自动将这些更改合并到你的工作分支中。
    它会更新你的本地副本中的远程跟踪分支（例如 origin/main），使你能够查看远程仓库的变化而不影响当前工作目录或本地分支。
    git merge：
    获取了远程的更改之后，git merge 会尝试将这些更改合并到当前所在的分支中。
    如果有冲突发生，Git 会提示你需要手动解决这些冲突。

git merge 和git rebase

merge 会对提交历史进行保留，很显然更适合多人协作开发的场景，因为如果出现问题也可以追溯到历史的每一次提交。

而 rebase 则是会让提交历史更加简洁易读，保持提交历史的线性结构，所以更适合个人开发和整理分支的情况。

将本地代码推送到github已创建的仓库

  1. git remote 查看是否关联远程仓库
  2. git remote add 地址别名 地址
  3. git remote -v 把地址别名和地址列出来
  4. 看分支名是否有冲突
  5. git branch -m main (-M强制) 切换远程分支
  6. ssh-keygen -t ed25519 -C “wwwyh123@github.com“生成ssh密钥
  7. cat ~/.ssh/id_ed25519.pub把该公钥复制到github上
