异常处理

throw

throw new Error('程序出错啦！')
程序会被中断，且抛出错误信息

try…catch…finally

用于捕获错误信息，将可能会出错的代码放到try中，

与throw 不同的是，catch捕获到错误后并不会中断程序，所以我们可以手动加一个return；catch中的参数，有一个属性message，可以提供错误信息。

作用域与作用域链

作用域分为全局作用域(.js文件或在<script>标签中)，局部作用域分为函数作用域或块级作用域
作用域链：** 本质上是底层的 变量查找机制 **,在函数被执行时，会优先查找当前作用域中查找变量，如果当前作用域查找不到，则会依次逐级查找父级作用域直到全局作用域

垃圾回收机制

什么是垃圾回收机制？
JS中内存的分配和回收都是自动完成的，内存在不使用的时候会被垃圾回收器自动回收
什么是内存泄漏？不再用到的内存，没有及时释放，就叫做内存泄漏
内存的生命周期是什么样的？

  1. 存分配、内存使用、内存回收
  2. 全局变量一般不会回收； 一般情况下局部变量的值, 不用了, 会被自动
    回收掉
    堆栈空间分配区别：
  3. 栈（操作系统）: 由操作系统自动分配释放函数的参数值、局部变量等，基本数据类型放到栈里面。
  4. 堆（操作系统）: 一般由程序员分配释放，若程序员不释放，由垃圾回收机制回收。复杂数据类型放到堆里面。
    两种常见的浏览器垃圾回收算法: 引用计数法 和 标记清除法

闭包

闭包=内层函数+外层函数的变量
闭包的作用？

  1. 封闭数据，实现数据私有，外部也可以访问函数内部的变量
  2. 闭包很有用，因为它允许将函数与其所操作的某些数据（环境）关联起来
    闭包可能引起的问题？
    内存泄漏
    常见闭包形式:外部可以访问使用函数内部的变量 封闭数据

事件循环

  1. js是单线程，防止代码阻塞，我们把代码（任务）分为同步和异步(分为宏任务和微任务)
  2. 同步代码给js引擎执行，异步代码交给宿主环境
  3. 同步代码放入执行栈中，异步代码等待时机送入任务队列排队
  4. 执行栈执行完毕，会去任务队列看是否有异步任务，有就送到执行栈执行，反复循环查看执行，这个过程就是事件循环
    宏任务：由宿主（浏览器，Node）发起的；script,网络请求(Ajax/Fetch),setTimeout()/setInterval()
    微任务：由js引擎发起的任务；Promise本身同步，then/catch回调函数是异步的；process.nextTick(node);Async/Await;Object.observe
    微任务优先于宏任务执行

    输出：2 3 6 p2 p1 1 4 5

    async函数里面的都是同步的
    await后面的都是微任务

Number

直接赋值，深浅拷贝

赋值没有创建新对象，仅仅是拷贝了原对象的地址。

浅拷贝是创建一个新对象，这个对象仅对原对象的属性进行拷贝。
属性值是基本类型时，拷贝的是原数据，属性值是引用类型时，拷贝的是地址。因此，如果原对象的属性有引用类型数据，无论修改新对象还是原对象的引用类型数据，另一个都会随之改变。

深拷贝也是创建一个新对象，但不仅对原对象的属性进行拷贝，还在堆内存中开辟一个新的地址用来存储新对象，所以新对象和原对象没有关联，修改其中一个另一个不会变化。
首先浅拷贝和深拷贝只针对引用类型

浅拷贝

常见方法：

  1. 拷贝对象：Object.assgin() / 展开运算符 {...obj} 拷贝对象
  2. 拷贝数组：Array.prototype.concat() 或者 [...arr]

    如果是简单数据类型拷贝值，引用数据类型拷贝的是地址 (简单理解： 如果是单层对象，没问题，如果有多层就有问题)

深拷贝

场景：

  1. 通过递归实现深拷贝
    想实现深拷贝用到函数递归，如果遇到普通类型，直接调用递归即可。如果遇到数组，判断出是数组类型后再次调用该递归函数即可。如果遇到对象，同理。但是一定要先数组，后对象(数组 instanceof Object true)。
  2. lodash/cloneDeep
    const o = _.cloneDeep(obj)
  3. 通过JSON.stringify()实现
  • 原理：JSON.stringify(obj)将对象转化为字符串（基本数据类型，在栈中开辟空间），JSON.parse()又将字符串转化为对象（引用数据类型，在堆中开辟空间）此时是新开辟的空间，与上面的毫无关联。
    const o = JSON.parse(JSON.stringify(obj))

防抖和节流(性能优化)

防抖

单位时间内，频繁的触发事件，只执行最后一次；（举例：王者回城，如果过程中被打断，需要再次触发，重新回城）
使用场景：搜索框搜索输入，手机号，邮箱验证输入监测

需要让鼠标移动时，500ms后再＋1

  • 实现的两种方式
    • 利用lodash的_.debounce(事件，时间)
      box.addEventListener('mousemove', _.debounce(move, 500))
    • 手写（核心：利用定时器实现）
      思路：
      1. 声明一个定时器变量
      2. 当鼠标每次滑动都先判断是否有定时器了，如果有定时器就先清除以前的定时器
      3. 如果没有定时器则开启定时器，记得存到变量里
      4. 在定时器里面调用要执行的函数 注意
        这里需要return一个函数，否则，页面一加载出来 debounce(move, 500) 就会被执行，并且返回值是 undefined ,所以每次触发mousemove 时就会调用undefined，即什么也不调用。反之，如果我们return一个函数，那么相当于box.addEventListener('mousemove', return的函数)，就可以正常执行

节流

单位时间内，频繁的触发事件，只执行一次（举例：王者放技能）
使用场景：鼠标移动mousemove，页面尺寸缩放resize，滚动条滚动scroll等

  • 实现的两种方式
    • 利用lodash的_.throttle(事件，时间)
      box.addEventListener('mousemove', _.throttle(move, 500))
    • 手写（核心：利用定时器实现）
      思路：
      1. 声明一个定时器变量
      2. 当鼠标每次滑动都先判断是否有定时器了，如果有定时器就不重开定时器
      3. 如果没有定时器则开启定时器，记得存到变量里
      • 定时器里面调用执行函数
      • 定时器里面要清空定时器 注意！ 在定时器中关闭定时器不能用clearTimeout 而是timer = 0
        节流的综合案例：打开一个视频仍保持上次观看的页面

原型

  • 每个函数都有prototype属性，称之为原型，因为这个属性的值是个对象，也称为原型对象
  • 作用：1. 存放一些属性和方法 2. 在js中实现继承

    原型链：对象都有__proto__属性，这个属性指向它的原型对象，原型对象也是对象，也有__proto__属性，指向原型对象的原型对象，这样一层层形成的链式结构称为原型链，最顶层找不到则返回null
