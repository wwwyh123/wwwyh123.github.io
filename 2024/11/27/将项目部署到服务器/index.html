将项目部署到服务器

Created2024-11-27|Updated2024-11-27
|Word Count:1.5k|Reading Time:4mins|Post Views:

大致步骤

  1. 本机打开windos+R 打开mstsc /admin,点击确定，出现

    将自己买的服务器的公网ip填进去，点击连接。就可以进入购买的服务器的windows操作系统。注意这里的用户名是Administer,试了root和administer都不对

  2. 在该windows中安装nginx。
    http://nginx.org/en/download.html
    下载后解压即可。
    具体操作步骤：
    https://blog.csdn.net/qq_31762741/article/details/121796712
    打开文件夹，cmd 中输入start nginx以启动Nginx
    如果窗口闪了一下，在浏览器中打开http://localhost:80,则表示安装成功。

  3. 修改nginx配置
    只需要修改以下两项

    listen 80;表示你的端口号 server_name 你买的服务器的Ip root html; 这个指令指定了请求的根目录。在这个例子中，root html; 意味着当Nginx接收到请求时，它会在服务器的文件系统中查找一个名为 html 的目录，并将这个目录作为请求的根目录。 需要注意的是，这里的 html 目录是相对于Nginx配置文件中定义的根路径（例如，通过 prefix 指令或编译时指定的默认路径）或者是绝对路径（如果 html 前有 /，表示从根文件系统开始）。 index index.html index.htm; 这个指令指定了当请求是一个目录而非文件时，Nginx 应该尝试返回哪个文件作为默认页面。在这个例子中，index index.html index.htm; 指示Nginx，如果请求的是一个目录（例如，仅仅请求了 / 而没有指定具体文件），则Nginx会尝试依次查找该目录下的 index.htm 文件。 综上所述，这段配置的目的是告诉Nginx，当接收到对根URL（/）的请求时，应该在服务器的 html 目录下查找资源，如果请求的是一个目录，则尝试返回该目录下的 index.html 文件。

  4. 修改完在此路径下的cmd 中输入nginx -s reload

  5. 配置安全组
    在你的服务器实例上配置一个以你的端口号为端口范围的入方向的配置规则。

  6. 在防火墙中设置一个入站规则
    只需要改如图的配置即可。
    最后，在任意ip输入你的服务器公网ip+端口号就可以访问！

在Windows电脑上启动并配置SSH服务

SSH全称secure shell，安全外壳协议（安全的shell），是一个计算机网络协议（默认端口号为22）。通过ssh协议可以在客户端安全（提供身份认证、信息加密）的远程连接LInux服务器或其他设备。
SSH远程连接之后，就可以像操作本地的机器一样操作远程机器。当需要操控的机器不在本地时就可以使用ssh协议远程连接操控。
原文链接：https://blog.csdn.net/m0_51720581/article/details/131796669
1.按下Win+R键输入services.msc,在服务列表中查找“OpenSSH SSH Server”服务.如果没找到就说明没有安装。
安装SSH服务：
搜索可选功能->添加可选功能->找到OpenSSH服务器->添加。
安装后，重复1的操作，点击启动服务，并且右击->属性->启动类型->’自动’->确定。
2.设置防火墙规则以允许SSH连接。在“控制面板”中的“Windows Defender 防火墙”设置中，允许端口22（SSH默认端口）的入站连接。

利用Workbench远程连接(连接linux实例)

就可以进入远程操作系统界面了

确保Workbench已正确安装并配置：
Workbench应已安装在你的本地计算机上，并且能够成功连接到阿里云的远程服务器。
准备要上传的本地项目：
确定你要上传的本地项目文件夹，并确保其中包含所有必要的文件和子文件夹。
二、上传项目
打开Workbench：
启动Workbench，并登录到你的阿里云远程服务器。
打开文件树：
在Workbench的顶部菜单栏中，找到并点击“文件”选项，然后选择“打开新文件树”。
导航到目标位置：
在文件树中，导航到你希望上传项目文件的远程服务器目录。
上传文件：
右键点击目标目录，选择“上传文件”。
在弹出的文件选择对话框中，浏览本地计算机，找到并选择你要上传的项目文件夹（注意，如果Workbench支持直接上传文件夹，可以直接选择文件夹；如果不支持，则需要逐个上传文件夹内的文件）。
选择文件后，点击“打开”或“确定”开始上传。
等待上传完成：
Workbench会显示上传进度，你可以等待上传完成。
上传完成后，你可以在远程服务器的目录中看到已经上传的项目文件。
三、注意事项
文件大小限制：
需要注意Workbench对上传文件的大小限制。如果项目文件过大，可能需要考虑使用其他工具（如WinSCP、FTP客户端等）进行上传。
网络稳定性：
在上传过程中，确保网络连接稳定，以避免上传失败或文件损坏。
权限问题：
确保你在远程服务器上具有足够的权限来上传文件到目标位置。如果权限不足，可能需要联系服务器管理员进行权限调整。
文件完整性：
上传完成后，建议检查上传的文件是否完整，并验证其功能和性能是否与本地版本一致。
